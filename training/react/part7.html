7.1 Beyond useState: Managing Real App State

As apps grow, state management gets more involved — you have more state, state shared across many components, and more complex logic — and React provides tools beyond basic useState to handle this cleanly. The key challenges: sharing state across components that are far apart in the tree (passing props down many levels gets unwieldy — 'prop drilling'), organising complex state and logic, and reusing stateful logic across components. React's answers include more hooks (useReducer for complex state logic, useRef for values that persist without re-rendering, useContext for shared state, and others), the Context API (for sharing state across the tree without prop drilling — e.g. the current user, theme, or auth status that many components need), and custom hooks (extracting reusable stateful logic into your own hooks). You don't need every advanced tool for every app, but understanding how to manage state well as an app grows — sharing it sensibly, organising it cleanly, and reusing logic — is what separates a tidy, scalable React app from a tangled one, and it's a real part of professional React work.

7.2 Context, Hooks, and Custom Hooks (Practical)

STATE MANAGEMENT & MORE HOOKS
  THE RULES OF HOOKS (always follow):
    · only call hooks at the TOP LEVEL of a component (not in loops/conditions/nested fns)
    · only call hooks in React components or custom hooks
  SHARING STATE — CONTEXT (avoid prop drilling for app-wide data: user, theme, auth):
    const ThemeContext = createContext();
    <ThemeContext.Provider value={theme}> ...app... </ThemeContext.Provider>  // provide
    const theme = useContext(ThemeContext);   // consume anywhere below — no prop drilling
  MORE HOOKS:
    · useReducer: for complex state logic (state + a reducer fn — like a mini Redux)
    · useRef: a mutable value that persists across renders WITHOUT causing re-render
      (also: reference a DOM element)
    · useMemo / useCallback: memoise expensive values/functions (performance — use when needed)
  CUSTOM HOOKS (reuse stateful logic — your own hook):
    function useFetch(url) { /* useState + useEffect inside */ return { data, loading, error }; }
    const { data, loading } = useFetch('/api/users');   // reuse the logic anywhere
  WHEN APPS GET BIG: libraries like Zustand / Redux Toolkit (know they exist; Context+hooks
    cover a lot)
KEY: follow the rules of hooks; Context shares state (no prop drilling); custom hooks reuse logic.

First, the rules of hooks (always follow): only call hooks at the top level of a component (not in loops, conditions, or nested functions) and only in components or custom hooks. Context shares state across the tree without prop drilling — create a context, wrap part of your app in its Provider value={...}, and useContext to consume it anywhere below (ideal for app-wide data like the current user, theme, or auth). More hooks: useReducer (complex state logic via a reducer — like a mini Redux), useRef (a value that persists across renders without causing a re-render, or a DOM reference), and useMemo/useCallback (memoising for performance — when needed). Custom hooks extract reusable stateful logic into your own hook (useFetch(url) bundling the fetch pattern). And for big apps, libraries like Zustand or Redux Toolkit exist (though Context + hooks cover a lot).

7.3 Clean, Scalable State Management

Managing state well is what keeps a growing React app clean and scalable rather than tangled — a mark of real React competence. The principles: keep state as local as possible (in the component that needs it), lift it up when shared (to the nearest common parent), and reach for Context when data is needed widely across the tree (avoiding deep prop drilling) — choosing the simplest approach that works rather than over-engineering. Use custom hooks to extract and reuse stateful logic (a useFetch, a useForm, a useToggle), which keeps components clean and logic DRY — a powerful, professional pattern. Follow the rules of hooks religiously (breaking them causes baffling bugs). And know that for large apps there are dedicated state-management libraries (Zustand, Redux Toolkit), but that modern React (hooks + Context) handles a great deal without them, so you reach for libraries only when genuinely needed. The deeper skill is judgement — where should each piece of state live? what's shared and how? what logic can be reused? — which comes with practice and is exactly what makes an app maintainable as it grows. Understanding hooks, Context, and custom hooks, and managing state thoughtfully, demonstrates you can build real, scalable React apps, not just toy components — well-managed app state is what separates a professional React codebase from a messy one.

CHECKPOINT

Manage app state well: use Context to share data app-wide without prop drilling (e.g. a theme or user), write a custom hook to reuse stateful logic (e.g. a useFetch or useToggle), and apply the rules of hooks and sensible state placement (local, lifted, or Context). Know that useReducer and state libraries exist for bigger needs. Clean, scalable state management — the right state in the right place, reusable logic in custom hooks — is what separates a professional React app from a tangled one.
