2.1 What Is CSS?

CSS (Cascading Style Sheets) controls how HTML looks: colours, fonts, spacing, layout, animation. A CSS rule has three parts: a selector (which elements to style), properties (what to change), and values (what to change it to).

/* selector { property: value; } */
h1 {
  color: #C8A04B;        /* gold text */
  font-family: Georgia;  /* serif font */
  font-size: 40px;
}

Text between /* and */ is a CSS comment. Connect CSS to HTML by adding one line inside <head>: <link rel="stylesheet" href="styles.css">. Keeping CSS in its own file is professional practice.

2.3 The Box Model: The Most Important Idea in CSS

Every element on a page is a rectangular box made of four layers, from inside out: content (the text or image), padding (space inside the border), border (the visible edge), and margin (space outside, pushing neighbours away). When a layout looks wrong, the answer is almost always in the box model.

.card {
  width: 300px;              /* content width */
  padding: 20px;             /* breathing room inside */
  border: 1px solid #ddd;    /* thin gray edge */
  border-radius: 8px;        /* rounded corners */
  margin: 16px;              /* space between cards */
  box-shadow: 0 2px 8px rgba(0,0,0,0.08);  /* soft shadow */
}
PRO HABIT

Start every stylesheet with: * { box-sizing: border-box; } This makes padding and border count INSIDE the width you set, which is how every sane developer expects boxes to behave.

2.4 Flexbox: Arranging Things in a Row or Column

Flexbox solves the historically painful problem of aligning things. Put display: flex on a parent, and its children line up obediently. This single practical teaches you 90% of what you will use daily:

FILEpart2-css/flexbox.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Flexbox Practice</title>
  <style>
    .nav {
      display: flex;
      justify-content: space-between; /* push items apart horizontally */
      align-items: center;            /* centre them vertically */
      background: #0A0A0A;
      color: white;
      padding: 16px 32px;
    }
    .nav a { color: white; margin-left: 24px; text-decoration: none; }
    .cards {
      display: flex;
      gap: 16px;            /* space between children */
      flex-wrap: wrap;      /* wrap to next line on small screens */
      padding: 32px;
    }
    .card {
      flex: 1;              /* share available space equally */
      min-width: 220px;
      padding: 24px;
      border: 1px solid #ddd;
      border-radius: 8px;
    }
  </style>
</head>
<body>
  <nav class="nav">
    <strong>MyBrand</strong>
    <div>
      <a href="#">Home</a><a href="#">Jobs</a><a href="#">Contact</a>
    </div>
  </nav>
  <div class="cards">
    <div class="card"><h3>Card One</h3><p>Resize the window and watch me adapt.</p></div>
    <div class="card"><h3>Card Two</h3><p>Flexbox shares space between us.</p></div>
    <div class="card"><h3>Card Three</h3><p>gap keeps us politely apart.</p></div>
  </div>
</body>
</html>
TRY THIS

Change justify-content to center, then flex-end. Change flex-direction on .cards to column. Predict before you save.

2.5 CSS Grid: Two-Dimensional Layouts

Flexbox arranges items along one line; Grid arranges them in rows AND columns at once — perfect for photo galleries, dashboards, and job listing pages.

.gallery {
  display: grid;
  grid-template-columns: repeat(3, 1fr);  /* three equal columns */
  gap: 16px;
}
/* On phones, collapse to one column */
@media (max-width: 600px) {
  .gallery { grid-template-columns: 1fr; }
}

That @media block is a media query — the foundation of responsive design. It says: "when the screen is 600 pixels wide or less, apply these rules instead." Mobile users are the majority of web traffic worldwide; every page you build must work on a phone.

2.6 Checkpoint Project: Style Your Profile Page

Create styles.css next to your Part 1 profile.html, link it in the head, and meet these requirements:

A dark header with flexbox navigation that stays readable on a phone.

Project cards arranged with Grid: three columns on desktop, one on mobile.

A consistent colour palette of no more than three colours, and at most two fonts (try Georgia for headings, Arial or system fonts for body).

Styled form inputs with padding, borders, and a button that changes colour on :hover.

Test on your phone: run Live Server, find your computer’s IP address, and visit it from your phone on the same Wi-Fi — or simply narrow the browser window.

DESIGN PRINCIPLE

Restraint reads as premium. Generous white space, few colours, clear hierarchy. A simple page done cleanly beats a busy page every time.
