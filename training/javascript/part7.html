7.1 Why Databases?

Your Part 6 API loses everything when the server restarts, because data lived in a variable in memory. A database is a program built purely to store, organise, search, and protect data permanently. The dominant family is relational databases, which store data in tables (like disciplined spreadsheets) and speak a language called SQL — Structured Query Language. SQL has been a core developer skill for fifty years and appears in virtually every backend job description.

We will use SQLite: a complete, real SQL database that lives in a single file — zero installation, zero configuration, and the same SQL skills transfer directly to PostgreSQL and MySQL in production jobs.

7.2 SQL in One Sitting (Practical)

Node ships with SQLite support built in (Node 22+, via node:sqlite). Create this file in a new part7-capstone folder and run it with node sql-practice.js:

FILEpart7-capstone/sql-practice.js
import { DatabaseSync } from "node:sqlite";
const db = new DatabaseSync("practice.db");   // creates the file if absent
// CREATE TABLE: define the shape of your data
db.exec(`
  CREATE TABLE IF NOT EXISTS jobs (
    id      INTEGER PRIMARY KEY AUTOINCREMENT,
    title   TEXT NOT NULL,
    company TEXT NOT NULL,
    salary  INTEGER DEFAULT 0,
    type    TEXT DEFAULT 'full-time'
  )
`);
// INSERT: add rows. The ? placeholders prevent SQL injection attacks —
// NEVER glue user input directly into a SQL string.
const insert = db.prepare(
  "INSERT INTO jobs (title, company, salary, type) VALUES (?, ?, ?, ?)"
);
insert.run("Frontend Developer", "RemoteFirst", 250000, "full-time");
insert.run("Data Analyst", "InsightCo", 350000, "full-time");
insert.run("Virtual Assistant", "GlobalDesk", 150000, "part-time");
// SELECT: read rows
const all = db.prepare("SELECT * FROM jobs").all();
console.log("All jobs:", all);
const wellPaid = db.prepare(
  "SELECT title, salary FROM jobs WHERE salary >= ? ORDER BY salary DESC"
).all(200000);
console.log("₦200k+:", wellPaid);
// UPDATE and DELETE
db.prepare("UPDATE jobs SET salary = ? WHERE id = ?").run(280000, 1);
db.prepare("DELETE FROM jobs WHERE id = ?").run(3);
// Aggregate functions
const stats = db.prepare(
  "SELECT COUNT(*) AS total, AVG(salary) AS avgSalary FROM jobs"
).get();
console.log(stats);

UPDATE ... SET ... WHEREChange matching rows (never forget WHERE!)
DELETE FROM ... WHERERemove matching rows (never forget WHERE!)

7.3 THE CAPSTONE: Remote Job Board — A Complete Full-Stack Application

This is the project that goes at the top of your CV. It combines every single skill from Parts 1–7: semantic HTML, styled responsive CSS, DOM manipulation, fetch, async/await, an Express REST API, and a SQL database. Build it in three files inside part7-capstone.

Step 1 — The Server With a Real Database

Run npm init -y then npm install express in the folder, add "type": "module" to package.json, and create:

FILEpart7-capstone/server.js
import express from "express";
import { DatabaseSync } from "node:sqlite";
const app = express();
const db = new DatabaseSync("jobboard.db");
db.exec(`
  CREATE TABLE IF NOT EXISTS jobs (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    title     TEXT NOT NULL,
    company   TEXT NOT NULL,
    salary    INTEGER DEFAULT 0,
    type      TEXT DEFAULT 'full-time',
    createdAt TEXT DEFAULT CURRENT_TIMESTAMP
  )
`);
app.use(express.json());
app.use(express.static("public"));   // serves the front end
// READ all (optional ?type= and ?search= filters)
app.get("/api/jobs", (req, res) => {
  const { type, search } = req.query;
  let sql = "SELECT * FROM jobs WHERE 1=1";
  const params = [];
  if (type)   { sql += " AND type = ?";                 params.push(type); }
  if (search) { sql += " AND (title LIKE ? OR company LIKE ?)";
                params.push(`%${search}%`, `%${search}%`); }
  sql += " ORDER BY createdAt DESC";
  res.json(db.prepare(sql).all(...params));
});
// CREATE
app.post("/api/jobs", (req, res) => {
  const { title, company, salary, type } = req.body;
  if (!title?.trim() || !company?.trim()) {
    return res.status(400).json({ error: "title and company are required" });
  }
  const info = db.prepare(
    "INSERT INTO jobs (title, company, salary, type) VALUES (?, ?, ?, ?)"
  ).run(title.trim(), company.trim(), Number(salary) || 0, type || "full-time");
  const job = db.prepare("SELECT * FROM jobs WHERE id = ?")
               .get(info.lastInsertRowid);
  res.status(201).json(job);
});
// DELETE
app.delete("/api/jobs/:id", (req, res) => {
  const info = db.prepare("DELETE FROM jobs WHERE id = ?")
                .run(Number(req.params.id));
  if (info.changes === 0) return res.status(404).json({ error: "Not found" });
  res.status(204).end();
});
app.listen(3000, () =>
  console.log("Job Board running at http://localhost:3000"));

FILEpart7-capstone/public/app.js
const form = document.querySelector("#jobForm");
const list = document.querySelector("#jobList");
const search = document.querySelector("#search");
async function loadJobs() {
  const q = search.value.trim();
  const url = "/api/jobs" + (q ? "?search=" + encodeURIComponent(q) : "");
  const res = await fetch(url);
  const jobs = await res.json();
  list.innerHTML = jobs.length === 0
    ? "<p>No jobs found. Post the first one!</p>"
    : "";
  jobs.forEach(job => {
    const row = document.createElement("div");
    row.className = "job";
    row.innerHTML = `
      <div>
        <strong>${job.title}</strong> · ${job.company}
        <span class="badge">${job.type}</span><br>
        <span class="salary">₦${Number(job.salary).toLocaleString()}/mo</span>
      </div>`;
    const del = document.createElement("button");
    del.className = "del";
    del.textContent = "✕";
    del.addEventListener("click", async () => {
      await fetch("/api/jobs/" + job.id, { method: "DELETE" });
      loadJobs();
    });
    row.appendChild(del);
    list.appendChild(row);
  });
}
form.addEventListener("submit", async (e) => {
  e.preventDefault();
  const body = {
    title:   document.querySelector("#title").value,
    company: document.querySelector("#company").value,
    salary:  document.querySelector("#salary").value,
    type:    document.querySelector("#type").value,
  };
  const res = await fetch("/api/jobs", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(body),
  });
  if (res.ok) { form.reset(); loadJobs(); }
  else { alert((await res.json()).error); }
});
search.addEventListener("input", loadJobs);
loadJobs();

Capstone Upgrade Challenges (do at least two)

Add an "Apply" button per job that records applications in a second table (applications: id, jobId, applicantName, email).

Add an edit feature using the PUT method.

Add filter buttons (All / Full-time / Part-time / Contract) that use the ?type= query parameter.

Add server-side validation: salary must be a positive number; title at most 80 characters.

Add a simple admin password: POST and DELETE require a header "x-admin-key" that the server checks.
