3.1 Arrays and Objects: How Data Is Organised

Real programs work with collections and structured data, and JavaScript's two core data structures handle this. Arrays are ordered lists of values ([1, 2, 3], ['Ada', 'Grace']) — perfect for sequences of things (a list of users, products, tasks). Objects are collections of named properties ({ name: 'Ada', age: 36 }) — perfect for representing a thing with multiple attributes (a user with a name, email, and role). Almost all real data is some combination of these: a list of user objects, an object containing arrays, and so on (this nested structure is also exactly what JSON — the universal data format of the web and APIs — looks like, so understanding arrays and objects means understanding JSON too). Mastering how to create, access, and manipulate arrays and objects is fundamental, because working with data is most of what programs do — and JavaScript provides exceptionally powerful, expressive methods for doing so, which are a joy to learn and a core part of fluency.

3.2 Arrays, Objects, and Their Methods (Practical)

DATA STRUCTURES & THEIR METHODS
  ARRAYS (ordered lists):
    const nums = [1, 2, 3];
    nums.length · nums[0] (access) · nums.push(4) (add) · nums.pop() (remove last)
    POWERFUL ARRAY METHODS (master these — the heart of JS data work):
      .map(n => n * 2)            // transform each -> new array [2,4,6]
      .filter(n => n > 1)         // keep matching -> [2,3]
      .find(n => n === 2)         // first match -> 2
      .forEach(n => console.log(n))   // do something with each
      .reduce((sum, n) => sum + n, 0) // combine into one value -> 6
      .sort(), .includes(), .some(), .every()
  OBJECTS (named properties):
    const user = { name: "Ada", age: 36, skills: ["JS"] };
    user.name · user["age"] (access) · user.email = "..." (add/change)
    Object.keys(user) · Object.values(user) · Object.entries(user)
  NESTING & JSON: arrays of objects (a list of users) = the shape of most real data & JSON
    JSON.parse(str) / JSON.stringify(obj)   // convert between JSON text and JS
KEY: arrays (lists) + objects (things) + array methods (map/filter/reduce) = working with data.

Arrays: create, access by index (nums[0]), push/pop to add/remove — but the real power is the array methods, which you should master: .map() (transform each item into a new array), .filter() (keep matching items), .find() (first match), .forEach() (do something with each), .reduce() (combine into one value), plus sort, includes, some, every. These expressive methods are the heart of JavaScript data work and replace clunky manual loops. Objects: named properties accessed with .name or ["name"], added/changed by assignment, with Object.keys/values/entries to inspect them. And nesting — arrays of objects (a list of users) is the shape of most real data and of JSON, which you convert with JSON.parse/JSON.stringify. Arrays, objects, and array methods are how you work with data.

3.3 Transforming Data Fluently

The array methods — especially map, filter, and reduce — deserve real focus because transforming collections of data is so central to modern JavaScript (and to React, Node, and almost everything you'll build). Instead of writing manual loops to process lists, you express transformations declaratively: 'take this list of products and map it to their names', 'take these users and filter to the active ones', 'take these prices and reduce them to a total'. This functional, method-chaining style (users.filter(u => u.active).map(u => u.name)) is clean, readable, and ubiquitous in professional JavaScript — you'll see and use it constantly, especially in React where rendering lists of data is array.map() everywhere. Becoming fluent with these methods — reaching naturally for map to transform, filter to select, find to locate, reduce to aggregate — is a hallmark of someone comfortable in JavaScript, and it makes working with the data that flows through every app feel natural rather than laborious. Combined with understanding objects (representing structured things) and JSON (the data format you'll constantly send and receive from APIs in Part 7), mastering data structures and their methods is core fluency. Demonstrating clean data manipulation — transforming, filtering, and aggregating real collections with array methods — shows the practical command of data that all real JavaScript work depends on.

CHECKPOINT

Manipulate real data: take a collection (an array of objects — like a list of products or users) and transform it with array methods — map to reshape, filter to select, find to locate, reduce to aggregate — chaining them where useful, plus access and update object properties, and convert to/from JSON. Fluency with arrays, objects, and methods like map/filter/reduce is core JavaScript skill that underpins React, Node, and everything you'll build.
