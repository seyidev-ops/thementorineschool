3.1 What Is JavaScript, and Where Does It Run?

JavaScript (JS) is a true programming language: it makes decisions, repeats actions, remembers things, and responds to events. It is the only language every web browser understands natively, and thanks to Node.js it also runs on servers — which is why JavaScript developers can build entire applications, front to back, with one language. It is consistently the most in-demand language in remote job listings worldwide.

Two Ways to Run JavaScript (Practical)

Way 1 — The browser console: open Chrome, press F12, click the "Console" tab, type the line below, and press Enter:

console.log("Hello from the console!");

Way 2 — Node.js in the terminal: create a file part3-js/hello.js containing that same line, then in the terminal run:

cd part3-js
node hello.js

console.log() prints things to the screen. It is your flashlight for the entire journey — when in doubt, log it out. For browser practicals in this part, use this HTML wrapper and put your JS between the <script> tags:

3.2 Variables: The Computer’s Memory

A variable is a labelled box where you store a value to use later. Modern JavaScript gives you two keywords:

let — a box whose contents can change later.

const — a box that is sealed after you fill it (constant). Professionals default to const and only use let when the value genuinely changes.

const myName = "Adaeze";     // text (a "string") — note the quotes
let age = 24;                 // a number — no quotes
let isLearning = true;        // true/false (a "boolean")
let salary;                   // declared but empty: undefined
console.log(myName);          // Adaeze
console.log("Age:", age);     // Age: 24
age = 25;                     // allowed: let can change
// myName = "Bola";           // ERROR: const cannot be reassigned
// Joining text with values: template literals (backtick quotes)
console.log(`My name is ${myName} and I am ${age} years old.`);

NAMING RULES

Variable names use camelCase (firstName, totalPrice), cannot start with a number, cannot contain spaces, and should describe their contents. monthlySalary is professional; x is not.

3.3 Operators: Doing Things With Values

// Arithmetic
console.log(10 + 3);   // 13
console.log(10 - 3);   // 7
console.log(10 * 3);   // 30
console.log(10 / 3);   // 3.333...
console.log(10 % 3);   // 1  (remainder — surprisingly useful)
console.log(2 ** 5);   // 32 (2 to the power of 5)
// Shorthand
let score = 10;
score += 5;            // same as score = score + 5  → 15
score++;               // add 1                       → 16
// Comparison (result is always true or false)
console.log(5 > 3);          // true
console.log(5 >= 5);         // true
console.log(5 === 5);        // true  (equal in value AND type)
console.log(5 === "5");      // false (number vs string)
console.log(5 !== 4);        // true  (not equal)
// Logical
console.log(true && false);  // false — AND: both must be true
console.log(true || false);  // true  — OR: at least one true
console.log(!true);          // false — NOT: flips the value

CRITICAL RULE

Always compare with === (triple equals), never == (double). Double equals performs sneaky type conversion (5 == "5" is true!) and causes bugs that take hours to find. A single = is assignment, not comparison at all.

3.4 Conditionals: Teaching the Computer to Decide

const hour = 14;
if (hour < 12) {
  console.log("Good morning!");
} else if (hour < 17) {
  console.log("Good afternoon!");
} else {
  console.log("Good evening!");
}
// Real-world example: grading
const examScore = 73;
let grade;
if (examScore >= 70)      grade = "A";
else if (examScore >= 60) grade = "B";
else if (examScore >= 50) grade = "C";
else                      grade = "F";
console.log(`Score ${examScore} = Grade ${grade}`);
// The ternary operator: a one-line if/else
const access = age >= 18 ? "granted" : "denied";
// switch: cleaner when checking one value against many options
const day = "Saturday";
switch (day) {
  case "Saturday":
  case "Sunday":
    console.log("Weekend! Time for a side project.");
    break;
  default:
    console.log("Workday. Code for 90 minutes anyway.");
}

3.5 Loops: Doing Things Repeatedly

Computers exist to repeat boring work flawlessly. Loops are how you command repetition.

// for loop: when you know how many times
for (let i = 1; i <= 5; i++) {
  console.log("Push-up number " + i);
}
// Reads as: start i at 1; keep going while i <= 5; add 1 each round.
// while loop: when you repeat until a condition changes
let savings = 0;
let week = 0;
while (savings < 100000) {
  week++;
  savings += 12500;     // save ₦12,500 per week
}
console.log(`Goal reached in ${week} weeks.`);
// Looping over an array (you will do this daily as a developer)
const skills = ["HTML", "CSS", "JavaScript"];
for (const skill of skills) {
  console.log("I am learning " + skill);
}
// break stops a loop early; continue skips one round
for (let n = 1; n <= 10; n++) {
  if (n === 4) continue;   // skip 4
  if (n === 8) break;      // stop completely at 8
  console.log(n);          // prints 1 2 3 5 6 7
}

Hands-On Practical: The Multiplication Master

// Prints the full 12 times table using a loop inside a loop
for (let a = 1; a <= 12; a++) {
  let row = "";
  for (let b = 1; b <= 12; b++) {
    row += (a * b) + "\t";   // \t is a tab character
  }
  console.log(row);
}

3.6 Functions: Reusable Machines

A function is a named block of code you write once and run as many times as you like. Functions take inputs (parameters), do work, and hand back an output (return value). They are the single most important building block in all of programming.

// Defining a function
function greet(name) {
  return "Welcome aboard, " + name + "!";
}
// Calling (using) it
console.log(greet("Chidi"));    // Welcome aboard, Chidi!
console.log(greet("Funke"));    // Welcome aboard, Funke!
// Multiple parameters and real logic
function calculatePay(hourlyRate, hours) {
  const gross = hourlyRate * hours;
  const tax = gross * 0.075;        // 7.5% tax
  return gross - tax;
}
console.log(calculatePay(5000, 40));  // 185000
// Default parameter values
function brew(drink = "coffee") {
  return "Brewing " + drink;
}
console.log(brew());          // Brewing coffee
console.log(brew("tea"));     // Brewing tea
// Arrow functions: the modern, shorter syntax (you will see these everywhere)
const add = (a, b) => a + b;
const square = n => n * n;
console.log(add(3, 4), square(5));   // 7 25

MENTAL MODEL

Think of a function as a blender: ingredients go in (parameters), the machine does its work (the body), a smoothie comes out (the return value). return immediately ends the function and hands back the result — code after return never runs.

3.7 Arrays: Ordered Lists of Anything

const jobs = ["Frontend Dev", "Backend Dev", "Data Analyst"];
console.log(jobs[0]);          // "Frontend Dev" — counting starts at ZERO
console.log(jobs.length);      // 3
jobs.push("DevOps Engineer");  // add to the end
jobs.unshift("Intern");        // add to the beginning
const last = jobs.pop();       // remove and return the last item
console.log(jobs.includes("Data Analyst"));  // true
console.log(jobs.indexOf("Backend Dev"));    // 2
// The Big Three array methods — learn these cold:
const prices = [1000, 2500, 4000, 800];
// map: transform every item → new array
const withVAT = prices.map(p => p * 1.075);
// filter: keep only items that pass a test → new array
const affordable = prices.filter(p => p < 3000);
// reduce: boil everything down to one value
const total = prices.reduce((sum, p) => sum + p, 0);
console.log(withVAT);     // [1075, 2687.5, 4300, 860]
console.log(affordable);  // [1000, 2500, 800]
console.log(total);       // 8300
// Sorting (numbers need a compare function)
const sorted = [...prices].sort((a, b) => a - b);  // [800,1000,2500,4000]

3.8 Objects: Describing Real-World Things

An array answers "what is in the list?"; an object answers "what are this thing’s properties?". Almost every piece of data you will ever receive from a server arrives as objects.

const job = {
  title: "Junior Frontend Developer",
  company: "RemoteFirst Ltd",
  salary: 250000,
  isRemote: true,
  skills: ["HTML", "CSS", "JavaScript"],
  // Objects can contain functions, called "methods"
  describe: function () {
    return `${this.title} at ${this.company}`;
  }
};
// Reading values — two equivalent ways
console.log(job.title);          // dot notation (preferred)
console.log(job["salary"]);      // bracket notation
// Changing and adding
job.salary = 300000;
job.location = "Anywhere";
console.log(job.describe());     // Junior Frontend Developer at RemoteFirst Ltd
// Arrays of objects: THE most common data shape in real work
const applicants = [
  { name: "Ada",  score: 85 },
  { name: "Bayo", score: 62 },
  { name: "Chi",  score: 91 },
];
const passed = applicants.filter(a => a.score >= 70);
const names = passed.map(a => a.name);
console.log(names);              // ["Ada", "Chi"]

this inside a method refers to the object the method belongs to — "this very object". You will meet it again in Part 4.

3.9 Checkpoint Project 1: The Number-Guessing Game

Your first complete program with logic, loops, functions, and randomness. Save as part3-js/guess.html and open with Live Server. (It uses the browser’s simple prompt() pop-up so we can focus purely on logic — Part 4 replaces pop-ups with real interfaces.)

FILEpart3-js/guess.html
<!DOCTYPE html>
<html lang="en">
<head><meta charset="UTF-8"><title>Guess the Number</title></head>
<body>
  <h1>Guess the Number (1–100)</h1>
  <p>Check the console for your game history.</p>
  <script>
    // Math.random() gives 0–0.999...; scale it to 1–100
    const secret = Math.floor(Math.random() * 100) + 1;
    const maxAttempts = 7;
    let attempts = 0;
    let won = false;
    while (attempts < maxAttempts && !won) {
      const input = prompt(`Attempt ${attempts + 1} of ${maxAttempts}. Your guess?`);
      const guess = Number(input);          // convert text → number
      attempts++;
      if (Number.isNaN(guess)) {
        alert("That is not a number. Try again!");
        attempts--;                          // don't punish typos
      } else if (guess === secret) {
        alert(`Correct! You won in ${attempts} attempts.`);
        won = true;
      } else if (guess < secret) {
        alert("Too low. Go higher.");
      } else {
        alert("Too high. Go lower.");
      }
      console.log(`Attempt ${attempts}: guessed ${guess}`);
    }
    if (!won) alert(`Out of attempts! The number was ${secret}.`);
  </script>
</body>
</html>

3.10 Checkpoint Project 2: Console Expense Tracker

Run with node expenses.js. This project mirrors exactly how real applications model data: an array of objects plus functions that operate on it.

FILEpart3-js/expenses.js
const expenses = [];
function addExpense(description, amount, category) {
  expenses.push({
    id: expenses.length + 1,
    description,
    amount,
    category,
    date: new Date().toLocaleDateString(),
  });
}
function totalSpent() {
  return expenses.reduce((sum, e) => sum + e.amount, 0);
}
function totalByCategory(category) {
  return expenses
    .filter(e => e.category === category)
    .reduce((sum, e) => sum + e.amount, 0);
}
function printReport() {
  console.log("=== EXPENSE REPORT ===");
  for (const e of expenses) {
    console.log(`#${e.id} [${e.category}] ${e.description}: ₦${e.amount}`);
  }
  console.log("Total spent: ₦" + totalSpent());
}
// --- Use the tracker ---
addExpense("Internet data", 5000, "utilities");
addExpense("Lunch", 2500, "food");
addExpense("Transport to meetup", 1500, "transport");
addExpense("Suya", 3000, "food");
printReport();
console.log("Food total: ₦" + totalByCategory("food"));

Try This (your final Part 3 exam): Add three functions: removeExpense(id) (hint: filter), biggestExpense() (hint: reduce or sort), and averageExpense(). If you can build these without looking anything up, you have genuinely internalised JavaScript fundamentals.
