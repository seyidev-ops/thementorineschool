1.1 What Is HTML?

HTML stands for HyperText Markup Language. It is not a programming language — it is a markup language, which means it describes structure, not behaviour. Think of building a house: HTML is the skeleton and walls, CSS (Part 2) is the paint and decoration, and JavaScript (Part 3 onwards) is the electricity and plumbing that makes things actually work.

Every web page you have ever visited — Google, Facebook, your bank — is built on HTML. When your browser loads a page, it reads an HTML file and draws what it describes.

1.2 Your First Web Page (Practical)

In VS Code, right-click the part1-html folder and choose New File. Name it index.html — exactly that, all lowercase.

Copy and paste the code below into the file, then press Ctrl+S (Cmd+S on Mac) to save.

Right-click anywhere inside the file and choose "Open with Live Server". Your browser opens with your first web page.

FILEpart1-html/index.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My First Web Page</title>
</head>
<body>
  <h1>Hello, World! I am becoming a software developer.</h1>
  <p>This is my very first web page, built with my own hands.</p>
</body>
</html>

Line-by-Line Breakdown

<!DOCTYPE html> — tells the browser "this is a modern HTML5 document". Always the first line.

<html lang="en"> — the root container of everything. lang="en" tells screen readers and search engines the page is in English.

<head> — information ABOUT the page (title, character set). Nothing here is visible on the page itself.

<meta name="viewport" ...> — makes the page display properly on phones. Include it in every page you ever build.

<title> — the text shown in the browser tab.

<body> — everything visible lives here.

<h1> and <p> — a top-level heading and a paragraph.

THE ANATOMY OF A TAG

Most HTML elements have an opening tag <p>, content, and a closing tag </p> with a forward slash. The browser reads <p>Hello</p> as "this is a paragraph containing the word Hello". Forgetting a closing tag is the number one beginner bug.

TagMeaningExample
<h1> to <h6>Headings, biggest to smallest. One <h1> per page.<h2>About Me</h2>

Practical: A Page Using Every Essential Tag

Create a new file tags-practice.html in the same folder, paste this, save, and open with Live Server:

FILEpart1-html/tags-practice.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Tag Practice</title>
</head>
<body>
  <h1>The Developer's Cookbook</h1>
  <p>Welcome to my collection of <strong>simple recipes</strong> for busy coders.</p>
  <hr>
  <h2>Jollof Rice Essentials</h2>
  <p>You will need the following <em>core ingredients</em>:</p>
  <ul>
    <li>Long-grain rice</li>
    <li>Tomatoes and peppers</li>
    <li>Onions and seasoning</li>
  </ul>
  <h2>Steps</h2>
  <ol>
    <li>Blend tomatoes, peppers and onions.</li>
    <li>Fry the blend in oil until it thickens.</li>
    <li>Add rice and stock, then cook on low heat.</li>
  </ol>
  <h2>Learn More</h2>
  <p>Read the full guide on
    <a href="https://www.allrecipes.com" target="_blank">AllRecipes</a>.
  </p>
  <button>Save This Recipe</button>
</body>
</html>
TRY THIS

Add a third recipe section with its own heading, an image using <img src="https://picsum.photos/400/200" alt="Food photo">, and a numbered list of at least four steps. Note: target="_blank" makes a link open in a new tab.

1.4 Attributes: Giving Tags Extra Information

Attributes live inside the opening tag and follow the pattern name="value". You have already used several: href tells a link where to go, src tells an image where its file is, and alt provides text if the image fails to load (and for visually impaired users). Two attributes are so important they get their own introduction:

id="something" — a unique name for ONE element on a page. Like a national ID number — no two elements may share it. JavaScript will use ids constantly.

class="something" — a reusable label that MANY elements can share. Like a uniform — everyone in class "card" can be styled the same way by CSS.

1.5 Semantic HTML: Writing Pages That Mean Something

You could build an entire site out of <div> boxes, but professionals use semantic tags — tags whose names describe their purpose. Search engines rank semantic pages higher, screen readers navigate them better, and other developers understand them faster.

Semantic TagPurpose
<header>Top section of a page or article — logo, title, navigation
<nav>Navigation links
<main>The unique main content of the page (one per page)
<section>A themed grouping of content, usually with a heading
<article>Self-contained content: a blog post, a job listing, a product card
<aside>Side content: related links, adverts, author bio
<footer>Bottom of page: copyright, contact, social links

1.6 Forms: Collecting Information From Users

Forms are how the web collects data — logins, registrations, searches, payments. Master them now; every project from Part 4 onwards uses them.

FILEpart1-html/form-practice.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Form Practice</title>
</head>
<body>
  <h1>Workshop Registration</h1>
  <form>
    <label for="fullname">Full Name</label><br>
    <input type="text" id="fullname" name="fullname"
           placeholder="e.g. Adaeze Okafor" required><br><br>
    <label for="email">Email Address</label><br>
    <input type="email" id="email" name="email" required><br><br>
    <label for="phone">Phone Number</label><br>
    <input type="tel" id="phone" name="phone"><br><br>
    <label for="track">Choose a Track</label><br>
    <select id="track" name="track">
      <option value="frontend">Front-End Development</option>
      <option value="backend">Back-End Development</option>
      <option value="fullstack">Full-Stack Development</option>
    </select><br><br>
    <p>Experience level:</p>
    <input type="radio" id="beginner" name="level" value="beginner" checked>
    <label for="beginner">Beginner</label><br>
    <input type="radio" id="intermediate" name="level" value="intermediate">
    <label for="intermediate">Intermediate</label><br><br>
    <input type="checkbox" id="newsletter" name="newsletter">
    <label for="newsletter">Send me the weekly newsletter</label><br><br>
    <label for="goals">Your learning goals</label><br>
    <textarea id="goals" name="goals" rows="4" cols="40"
              placeholder="Tell us what you want to achieve..."></textarea><br><br>
    <button type="submit">Register Now</button>
  </form>
</body>
</html>

Key Points

<label for="x"> connects to the input whose id is x — clicking the label focuses the input. Always pair them.

type="email" and required give you free validation: the browser refuses to submit an empty or malformed email.

Radio buttons sharing the same name form a group where only one can be selected.

Common input types: text, email, password, number, tel, date, checkbox, radio, file, color, range. Try swapping types and watch the browser change behaviour.

1.7 Tables and Media

Tables display rows and columns of data (never use them for page layout — that is CSS’s job).

<table border="1">
  <thead>
    <tr>
      <th>Skill</th>
      <th>Level</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>HTML</td>
      <td>Learning</td>
    </tr>
    <tr>
      <td>JavaScript</td>
      <td>Next up</td>
    </tr>
  </tbody>
</table>
<!-- Video and audio -->
<video src="demo.mp4" controls width="400"></video>
<audio src="podcast.mp3" controls></audio>
<!-- Embed a YouTube video -->
<iframe width="560" height="315"
  src="https://www.youtube.com/embed/VIDEO_ID"
  title="YouTube video" allowfullscreen></iframe>

<tr> is a table row, <th> a header cell, <td> a data cell. The <!-- ... --> syntax is an HTML comment — notes for humans that the browser ignores.

1.8 Checkpoint Project: Your Personal Profile Page

This is your first real-world deliverable — the seed of the portfolio you will deploy in Part 9. Build it entirely yourself, using the template below as a starting skeleton.

Requirements

A header with your name and a navigation bar linking to each section (hint: <a href="#about"> jumps to the element with id="about").

An About section with your photo, a short bio, and a list of your goals.

A Skills section using a table: skill name, level, and learning status.

A Projects section with at least two <article> cards (use placeholder text for now — you will fill these with real projects soon).

A Contact section with a working form: name, email, message, submit button.

A footer with the current year and links to your social profiles.

FILEpart1-html/profile.html (starter skeleton)
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Your Name — Software Developer</title>
</head>
<body>
  <header>
    <h1>Your Name</h1>
    <nav>
      <a href="#about">About</a> |
      <a href="#skills">Skills</a> |
      <a href="#projects">Projects</a> |
      <a href="#contact">Contact</a>
    </nav>
  </header>
  <main>
    <section id="about">
      <h2>About Me</h2>
      <img src="https://picsum.photos/200" alt="Profile photo">
      <p>Write 2–3 sentences about who you are and where you are going.</p>
    </section>
    <section id="skills">
      <h2>Skills</h2>
      <!-- Build your skills table here -->
    </section>
    <section id="projects">
      <h2>Projects</h2>
      <article>
        <h3>Project One</h3>
        <p>Short description.</p>
      </article>
      <!-- Add a second article -->
    </section>
    <section id="contact">
      <h2>Contact Me</h2>
      <!-- Build your contact form here -->
    </section>
  </main>
  <footer>
    <p>&copy; 2026 Your Name. Built with my own hands.</p>
  </footer>
</body>
</html>
