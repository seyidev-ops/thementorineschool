8.1 How Languages Actually Talk to Each Other

Real-world systems are polyglot: a JavaScript front end, a Python machine-learning service, a SQL database, perhaps a legacy PHP system — all cooperating. The secret is that languages rarely call each other directly. Instead, they communicate through neutral channels that every language understands:

HTTP + JSON APIs — by far the most common bridge. Any language can run a web server; any language can fetch JSON. Your JavaScript already speaks this fluently.

Embedded query languages — SQL lives as strings inside your JavaScript (you did this in Part 7).

Child processes — Node can launch a Python or shell script directly and read its output.

WebAssembly — compiled languages like C, C++ and Rust can run inside the browser, called from JavaScript.

8.2 JavaScript + Python: The Power Couple

Python dominates data analysis and AI; JavaScript dominates interfaces. Together they cover most modern software. Install Python from python.org (tick "Add to PATH" on Windows), then build a Python API that your JavaScript front end consumes.

Step 1 — A Python API With FastAPI

FILEpart8-integration/analytics.py
# Install once:  pip install fastapi uvicorn
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from statistics import mean, median
app = FastAPI()
# CORS: explicitly allow browser pages from other origins to call us
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_methods=["*"],
    allow_headers=["*"],
)
@app.get("/api/hello")
def hello():
    return {"message": "Hello from Python!", "language": "Python 3"}
@app.post("/api/analyze")
def analyze(payload: dict):
    numbers = payload.get("numbers", [])
    if not numbers:
        return {"error": "Send a 'numbers' array"}
    return {
        "count": len(numbers),
        "total": sum(numbers),
        "average": round(mean(numbers), 2),
        "median": median(numbers),
        "max": max(numbers),
        "min": min(numbers),
    }
# Run with:  uvicorn analytics:app --port 8000

Step 2 — A JavaScript Front End That Calls Python

FILEpart8-integration/dashboard.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8"><title>JS ↔ Python Dashboard</title>
  <style>
    body { font-family: Arial; max-width: 520px; margin: 40px auto; }
    textarea { width: 100%; height: 70px; }
    pre { background: #f3f0e9; padding: 16px; border-radius: 8px; }
  </style>
</head>
<body>
  <h1>Sales Analyzer</h1>
  <p>JavaScript interface · Python brain</p>
  <textarea id="data">12000, 45000, 30000, 8000, 52000, 31000</textarea><br>
  <button id="run">Analyze with Python</button>
  <pre id="out">Results appear here…</pre>
  <script>
    document.querySelector("#run").addEventListener("click", async () => {
      const numbers = document.querySelector("#data").value
        .split(",").map(n => Number(n.trim())).filter(n => !isNaN(n));
      const res = await fetch("http://localhost:8000/api/analyze", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ numbers }),
      });
      const stats = await res.json();
      document.querySelector("#out").textContent =
        JSON.stringify(stats, null, 2);
    });
  </script>
</body>
</html>

Start Python with uvicorn analytics:app --port 8000, open the HTML with Live Server, click the button — and watch two languages cooperate across an HTTP boundary. This exact architecture (JS interface, Python computation) powers countless AI products today.

WHAT IS CORS?

Browsers block pages from calling servers on a different origin (domain/port) unless that server explicitly permits it — a security feature called Cross-Origin Resource Sharing. The Python middleware above grants permission. When your fetch mysteriously fails with a CORS error in the console, this is what is happening.

Bonus: Node Launching Python Directly (No Server Needed)

FILEpart8-integration/run-python.js
import { execFile } from "node:child_process";
// Suppose script.py prints a result; Node captures it:
execFile("python", ["script.py", "42"], (error, stdout) => {
  if (error) return console.error("Python failed:", error.message);
  console.log("Python said:", stdout.trim());
});

8.3 JavaScript + PHP: Working With the Classic Web

PHP still runs a vast share of the web (WordPress alone is over 40% of all sites). Knowing how JavaScript talks to PHP makes you valuable for maintaining and modernising existing businesses’ sites. The bridge is identical: PHP produces JSON, JavaScript fetches it.

FILEpart8-integration/api.php
<?php
// Install PHP, then run:  php -S localhost:9000
header("Content-Type: application/json");
header("Access-Control-Allow-Origin: *");
$products = [
  ["id" => 1, "name" => "Ankara Notebook", "price" => 3500],
  ["id" => 2, "name" => "Dev Mug",         "price" => 5000],
];
echo json_encode($products);
?>
// JavaScript side — exactly the same fetch you already know:
const res = await fetch("http://localhost:9000/api.php");
const products = await res.json();
products.forEach(p => console.log(p.name, "₦" + p.price));

THE BIG INSIGHT

Notice your JavaScript did not change at all between Python and PHP. Once a back end speaks JSON over HTTP, your front-end skills work against ANY language — Java, C#, Go, Ruby, Rust. You learned one bridge and unlocked them all.

8.4 JavaScript + SQL: A Deeper Look

You already integrate SQL daily since Part 7 — SQL statements are strings embedded in JavaScript, executed by a database engine. Two professional rules to carry forever:

Always use parameterised queries (? placeholders). Building SQL by gluing user input into strings enables SQL injection — the attack behind many famous data breaches. db.prepare("...WHERE id = ?").get(userInput) is safe; string concatenation is not.

In bigger projects, consider an ORM (Object-Relational Mapper) like Prisma, which lets you write prisma.job.findMany() instead of raw SQL. Learn raw SQL first — ORMs make sense only once you know what they are hiding.

8.5 JavaScript + Compiled Languages: WebAssembly

WebAssembly (WASM) lets code written in C, C++, Rust or Go run inside the browser at near-native speed, with JavaScript as the conductor. Figma (C++), Photoshop on the web, and many games use it. As a junior you mainly need the concept: heavy computation can be compiled to WASM, and JavaScript loads and calls it like this:

// Conceptual example: loading a compiled WASM module
const { instance } = await WebAssembly.instantiateStreaming(
  fetch("math-engine.wasm")
);
// Call a function written originally in Rust or C:
console.log(instance.exports.fibonacci(40));

When an interviewer asks "how would you handle CPU-heavy work in the browser?", the professional answer is: a Web Worker for parallelism, WebAssembly for raw speed, or push it to a server-side service — and you now understand all three options.

8.6 Checkpoint Project: The Polyglot Dashboard

Combine the part: build one HTML/JS dashboard page with three panels, each fed by a different backend you have built:

Panel 1 — Jobs: fetched from your Node/Express + SQLite capstone API (Part 7).

Panel 2 — Statistics: salary list sent to your Python FastAPI analyzer; display average, median, max.

Panel 3 — Products: fetched from your PHP endpoint.
