3.1 One Language, Two Homes

JavaScript is the only language that runs natively in both homes — the browser and (via Node) the server. Master it once, use it everywhere: that is the full-stack developer's structural advantage. Work in part3-js, running files with node filename.js.

3.2 Variables, Types and Template Strings

FILEpart3-js/01-basics.js
const store = "MentorMarket";        // const: cannot be reassigned (default)
let itemsInCart = 0;                  // let: can change
const serviceFee = 0.05;              // 5%
const isOpen = true;

Strings, numbers, booleans — plus null/undefined for emptiness. The backtick template literal with ${...} is how professionals build text; you will type it daily.

3.3 Decisions and Loops

FILEpart3-js/02-logic.js
const balance = 8000;
const price = 5000;

if (price > balance) {
  console.log("Insufficient funds.");
} else {
  console.log(`Purchase approved. Change: \u20a6${balance - price}`);
}

// Loop over the cart
const cart = ["CV Review", "Maths Tutoring", "Logo Design"];
for (const item of cart) {
  console.log(`Packing: ${item}`);
}

Always compare with === and !== — the triple forms refuse sneaky type conversions that the double forms allow, and that refusal prevents real bugs.

3.4 Functions: Small Machines That Compose

FILEpart3-js/03-functions.js
const naira = (n) => `\u20a6${n.toLocaleString()}`;

const withFee = (price) => price * 1.05;                 // 5% service fee

const lineTotal = (price, qty) => withFee(price) * qty;

console.log(`2 tutoring sessions: ${naira(lineTotal(3500, 2))}`);

Three tiny machines plugging into each other. Professional codebases are thousands of these — each doing one thing, named honestly — not one giant scroll of instructions.

3.5 Arrays and Objects: The Shapes APIs Speak

All the JSON your server will ever send or receive is just two shapes nested: arrays (ordered lists) and objects (labelled records). And three array methods do most of the real work:

FILEpart3-js/04-shapes.js
const listings = [
  { id: 1, title: "CV Review",      price: 5000, category: "career" },
  { id: 2, title: "Maths Tutoring", price: 3500, category: "tutoring" },
  { id: 3, title: "Logo Design",    price: 8000, category: "design" },
];

// map: transform every item
const titles = listings.map(l => l.title);

// filter: keep some items
const affordable = listings.filter(l => l.price <= 5000);

// reduce: boil down to one value
const cartTotal = listings.reduce((sum, l) => sum + l.price, 0);

console.log({ titles, affordable, cartTotal });
WHY THIS MATTERS

“Take these database rows, reshape them, drop what's private, total the amounts” — that sentence IS map/filter/reduce, and it is most of back-end data handling.

3.6 Async: The Kitchen That Never Stands Still

Network calls and database queries are slow; JavaScript refuses to stand and wait. It starts the slow thing, keeps working, and resumes when the result arrives — written cleanly with async/await. You already used it in Part 1's fetch; now see it bare:

FILEpart3-js/05-async.js
const fakeDbQuery = (sql) =>
  new Promise(resolve => setTimeout(() => resolve([{ id: 1, title: "CV Review" }]), 1200));

const main = async () => {
  console.log("Querying...");
  const rows = await fakeDbQuery("SELECT * FROM listings");   // pause HERE only
  console.log("Rows:", rows);
};

main();
console.log("(The server keeps serving other users meanwhile)");

Run it and study the output order. await pauses one function, never the whole program — the single idea behind Node's famous ability to serve thousands of users on one modest machine.
