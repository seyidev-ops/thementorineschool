9.1 Git & GitHub: Your Work, Versioned and Visible

cd fullstack-journey
git init
printf "node_modules/\n.env\n*.db\n" > .gitignore
git add .
git commit -m "Parts 1-8: storefront, API, database, auth"
# create an empty repo on github.com, then:
git remote add origin https://github.com/YOUR-USERNAME/fullstack-journey.git
git push -u origin main

Commit small, commit often, write messages that say why. A steady green graph plus a documented capstone is the cheapest interview advantage in the industry. Verify on GitHub that node_modules, .env and .db files did not upload — secrets and bulk stay home.

9.2 Making Code Deployment-Ready

Three small changes make any project deployable. The API accepts the platform's port and exposes a health check:

FILEAPI server.js (adjust)
const PORT = process.env.PORT || 4000;
app.get("/health", (req, res) => res.json({ status: "ok" }));
app.listen(PORT, () => console.log(`API on :${PORT}`));

And the React app stops hard-coding localhost — Vite injects environment variables that start with VITE_:

FILEmentormarket/src/auth.js and App.jsx (adjust)
const API = import.meta.env.VITE_API_URL || "http://localhost:4000/api";

9.3 Deploying Both Halves (Practical)

Back end on Render (render.com, free tier): New ▸ Web Service → connect your GitHub repo → root directory = your API folder → build npm ci, start node server.js → set JWT_SECRET in the Environment tab → deploy. You receive a URL like https://mentormarket-api.onrender.com. (SQLite resets on redeploy on the free tier — acceptable for a portfolio; Render's free Postgres is the one-string upgrade when you want permanence.)

Front end on Netlify (netlify.com): Add new site → import from GitHub → base directory = part7-react/mentormarket, build npm run build, publish directory dist → add environment variable VITE_API_URL = https://mentormarket-api.onrender.com/api → deploy. Two URLs, two halves, one application — open the Netlify URL on your phone and buy something from yourself.

PRO HABIT

After every deploy, curl your API's /health from your phone's browser. Thirty seconds of paranoia beats three hours of “but it worked locally”.

9.5 The Job Hunt: Selling the Whole Stack

Your portfolio now argues for itself: a live product, a documented repo, a commit history showing months of honest, steady work. Package it — README with the architecture sketch, a screen-recording GIF of the buy flow, a five-line “what I'd build next”. Target roles titled Full-Stack Developer (JavaScript/Node), Software Engineer, or Front-End/Back-End roles (you qualify for both — that is the point). In interviews, whiteboard MentorMarket's login flow from input box to database row and back; candidates who can walk the whole stack are interviewed as engineers, not students. The Mentorine School's career pillars — CV positioning, portfolio review, mock interviews — take it from here.
