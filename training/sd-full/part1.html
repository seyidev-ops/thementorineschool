1.1 Why the Terminal Comes First

The terminal (also called the command line, shell, or console) is a text-based way to control your computer. Every professional tool you will meet — Git, Node, Python, Docker, deployment platforms — is operated from the terminal. Learning it first means nothing later feels foreign.

Windows users: use PowerShell (built in). Mac and Linux users: use Terminal. The commands below work on all three; where Windows differs, both versions are shown.

1.3 How a Computer Actually Runs Your Code

A computer's processor only understands machine code — raw numbers. Everything you write must be translated. There are two translation strategies, and knowing the difference explains half the software world:

  • Interpreted languages (JavaScript, Python): a program called an interpreter reads your file line by line and executes it immediately. Fast to write and test — you just run the file.
  • Compiled languages (C, C++, Rust, Go): a compiler translates the whole program into machine code first, producing an executable file. Faster to run, slower to build.

You will work mainly in interpreted languages because they are the fastest path to building real products. The mental model is identical everywhere: source code in, running program out.

1.5 Files, Paths, and Extensions — The Map of Your Machine

  • A path is an address: C:\Users\Seyi\Documents\dev-journey (Windows) or /Users/seyi/Documents/dev-journey (Mac/Linux).
  • A relative path starts from where you are: ./part1-terminal/hello.js means "from here, go into part1-terminal". You will use relative paths constantly in code.
  • Extensions declare a file's language: .js JavaScript, .py Python, .html web page, .css styles, .json data, .md documentation.
