5.1 What the DOM Is

When a browser loads HTML, it builds a live object model of the page in memory — the Document Object Model. JavaScript can read it, change it, and react to the user through it. Every interactive thing you have ever clicked on the web is JavaScript manipulating the DOM.

The pattern below — state, render, actions, events — is the same architecture React uses in Part 6. Learn it here by hand and React will feel obvious instead of magical.

Why This Architecture Matters

  • State → render: the page is always a drawing of your data. Fix the data, the page fixes itself. This is the core idea of every modern framework.
  • localStorage keeps the tasks after refresh — your first persistence, free, built into the browser.
  • Event delegation: one listener on the list handles every button inside it, even buttons created later. Interviewers love asking about this.

5.3 The DOM Toolkit

ToolWhat it does
document.querySelector("css")Find the first element matching a CSS selector
element.textContent / .innerHTMLRead or replace text / HTML inside an element
element.classList.add / .remove / .toggleSwitch CSS classes — how you show/hide and animate
document.createElement + appendChildBuild new elements from code
element.addEventListener("click", fn)Run a function when the user acts (click, input, submit, keydown)
event.preventDefault()Stop a form from reloading the page
input.valueRead what the user typed

CONCEPT

async/await is how JavaScript waits for slow things (networks, files) without freezing the page. Any function that uses await must be declared async. You will use this in every project for the rest of your career.

5.5 Checkpoint Project — Quiz App

Build quiz.html from scratch using the state → render → actions → events pattern:

  1. An array of at least 8 question objects: { question, options: [...], answer }.
  2. Show one question at a time with its options as buttons.
  3. Clicking an option highlights right/wrong, updates the score, and moves on after a second (hint: setTimeout(fn, 1000)).
  4. A final screen with score, percentage, and a Restart button.
  5. Persist the best score in localStorage and display it. Commit and push to GitHub.
CHECKPOINT

You can now build complete interactive apps with persistence and live data. That is the entire frontend craft in miniature — Part 6 scales it.
