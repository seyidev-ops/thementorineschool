0.1 What a Software Developer Actually Does

A software developer turns ideas into working products: websites, mobile apps, APIs, automation scripts, databases, and the systems that connect them. The job is not memorising syntax — it is solving problems by breaking them into small steps a computer can execute, then assembling those steps into something people use.

This guide trains you as a full-stack developer: someone who can build the part users see (the frontend), the engine behind it (the backend), the data layer (databases), and ship it to the real world (DevOps). Full-stack developers are the most employable beginners because they can deliver a complete product alone.

0.2 The Golden Rules of Learning to Code

  1. Type or paste, then run, then break. Every code block in this guide is meant to be executed, not read. After it works, change something on purpose and watch what happens — that is where real learning lives.
  2. Two hours a day beats fourteen on Saturday. Programming skill is built like muscle: frequent, consistent sessions. The 24-week plan assumes 2–4 focused hours daily.
  3. Error messages are instructions, not insults. Professionals see hundreds of errors a day. Each one tells you exactly where to look. You will learn to read them in Part 10.
  4. Build before you feel ready. You will never feel ready. Every part of this guide ends with a project precisely because building is the only proof of learning.
  5. Do not skip the terminal and Git. Beginners avoid them; professionals live in them. They are Parts 1 and 4 for a reason.

0.4 Setting Up Your Developer Workstation (Practical)

You need one tool installed today — the rest are installed exactly when each part needs them, so nothing here is wasted.

Install Visual Studio Code

  1. Go to code.visualstudio.com and download the installer for your operating system (Windows, macOS, or Linux). Run it with default options.
  2. Open VS Code. Press Ctrl+Shift+X (Cmd+Shift+X on Mac) to open Extensions.
  3. Install these two extensions by searching their names: Live Server (by Ritwick Dey) and Prettier — Code formatter.

Install Node.js (JavaScript engine)

  1. Go to nodejs.org and download the LTS version (the button on the left). Install with default options.
  2. Restart your computer after installing — this matters on Windows.

Install Python

  1. Go to python.org/downloads and download the latest Python 3 release.
  2. Windows only: on the first installer screen, tick the box "Add python.exe to PATH" before clicking Install. This single checkbox prevents weeks of confusion.

Install Git

  1. Go to git-scm.com/downloads and install for your OS. Accept every default option — they are all sensible.
  2. macOS alternative: open Terminal, type git --version, and macOS will offer to install it for you.

Verify Everything (Your First Terminal Commands)

Open a terminal: on Windows, search for "PowerShell"; on Mac, search for "Terminal". Type each line below and press Enter. Each should print a version number — if any says "not recognized", reinstall that tool.

CHECKPOINT

All three commands print version numbers, and VS Code opens with Live Server and Prettier installed. Your workstation is professional-grade. Welcome to Day 1.

0.5 Your Project Folder for the Whole Course

Create one folder that will hold everything you build for the next 24 weeks. In your Documents folder, create a folder named dev-journey. Inside it, you will create one sub-folder per part as you reach it (part1-terminal, part2-web, and so on). Open the dev-journey folder in VS Code now: File → Open Folder.
