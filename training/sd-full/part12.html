12.1 The Universal Bridge: HTTP + JSON

Real companies are polyglot: a JavaScript frontend, a Python data service, a PHP legacy system, a Go payments engine. They cooperate through one universal bridge — services expose HTTP APIs and exchange JSON. Because you already speak HTTP and JSON, you can integrate any language ever made. This part proves it three ways.

The lesson generalises: Go, Java, C#, Ruby — give each service an HTTP+JSON face and they all interoperate. This architecture has a name you will hear in interviews: services (or microservices when split very finely). Two final bridges worth knowing by name:

  • WebAssembly (WASM): compiles C, C++ and Rust to run INSIDE the browser at near-native speed — how Figma and Google Earth run on a web page.
  • Message queues (Redis, RabbitMQ): services drop jobs in a queue instead of calling each other directly — the pattern behind "your report will be emailed shortly".
