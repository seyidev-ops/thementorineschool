9.1 Why Your API Needs a Database

Your Part 8 API kept data in a JavaScript array — restart the server and everything vanishes. A database stores data on disk, survives restarts, handles thousands of users at once, and enforces rules about what valid data looks like. Two big families exist:

  • Relational (SQL): data in tables with rows and columns, queried with the SQL language. PostgreSQL, MySQL, SQLite. The default choice for most applications — and what most jobs ask for.
  • Document (NoSQL): data as JSON-like documents. MongoDB is the famous one. Flexible shape, popular with Node, best when data is naturally nested.

You will learn SQL hands-on with SQLite — a complete, serious SQL database that lives in a single file and is built into Node.js, so there is nothing to install. Everything you learn transfers directly to PostgreSQL and MySQL.

WARNING

An UPDATE or DELETE without a WHERE clause hits EVERY row in the table. Every experienced developer has a scar from this. Type the WHERE first.

9.3 SQL Injection — The Attack You Must Never Allow

Never build SQL by gluing user input into the string: "... WHERE name = '" + userInput + "'" lets an attacker type '; DROP TABLE students; -- and delete your data. The ? placeholders you used above (prepared statements / parameterised queries) make this attack impossible — the database treats input as data, never as commands. This is non-negotiable in every language and every job.

9.6 NoSQL in Five Minutes — and When to Choose What

In MongoDB the same student is a JSON document: { name: "Ada", scores: [82, 90], address: { city: "Abuja" } } — nesting is natural and the shape can vary per document. Operations mirror what you know: insertOne, find, updateOne, deleteOne.

Choose SQL when…Choose NoSQL when…
Data is structured and inter-related (users, orders, payments)Data is nested or varies in shape (content, logs, profiles)
You need strict integrity and transactionsYou need flexible, fast iteration on shape
Most jobs and most apps — the safe defaultSpecific fit — adopt when the data says so
