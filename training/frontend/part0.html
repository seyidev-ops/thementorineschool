0.1 Who This Guide Is For

This guide assumes you have never written a single line of code. If you can type, save a file, and open a web browser, you have every prerequisite you need. By the end, you will be able to build complete, professional, interactive websites and web applications — and present yourself credibly as a junior front-end developer ready for local or remote work.

The promise is simple: every concept is explained in plain English, every concept is immediately followed by a hands-on practical you can copy, paste, and run, and every part ends with a real-world project that belongs in your portfolio.

0.2 The Golden Rules of Learning to Code

Type it, then run it. Copy-paste gets you moving fast, but retype at least one practical per day by hand. Muscle memory is real.

Break things on purpose. After every working practical, change something and predict what will happen. This is how professionals actually learn.

Code every day. Ninety minutes daily beats ten hours on Saturday. Consistency rewires your brain; cramming does not.

Errors are information, not failure. A red error message is the computer telling you exactly where to look. Read it slowly, from the top.

Build, don't just watch. Tutorials feel productive but build nothing. Every project in this guide produces something real you can show an employer or a client.

0.4 Setting Up Your Developer Workstation

You need exactly three tools. All are free. Install them now, in this order.

Tool 1: Visual Studio Code (your code editor)

Go to https://code.visualstudio.com and click the big download button for your operating system (Windows, Mac, or Linux).

Run the installer. Accept all default options. On Windows, tick “Add to PATH” and “Open with Code” when offered.

Open VS Code. Click the Extensions icon on the left sidebar (four squares). Search for and install: Live Server (by Ritwick Dey), Prettier (code formatter), and ES7+ React snippets (you will need it from Part 7).

WHY THIS MATTERS Live Server lets you see your web page update instantly every time you save. Prettier keeps your code tidy automatically. Professionals use both daily.

Tool 2: Google Chrome (your testing browser)

Any modern browser works, but this guide uses Chrome because its built-in Developer Tools (DevTools) are the industry standard. Download it from google.com/chrome if you don't have it. Memorise this shortcut now: press F12 (or Ctrl+Shift+I on Windows, Cmd+Option+I on Mac) to open DevTools. You will live in this panel.

Tool 3: Node.js (the engine behind modern front-end tooling)

Even pure front-end developers need Node.js, because the modern toolchain — npm, Vite, Tailwind, React — runs on it.

Go to https://nodejs.org and download the version labelled LTS (Long Term Support).

Run the installer with all default options.

Verify it worked: open your terminal (Windows: press the Windows key, type “cmd”, press Enter. Mac: open the Terminal app) and type the commands below, pressing Enter after each.

node --version
npm --version

You should see version numbers like v22.x.x and 10.x.x. If you see “command not found”, restart your computer and try again — the installer needs a restart to register itself.

Create Your Project Folder

Keep everything for this course in one folder. In your terminal, type:

mkdir frontend-journey
cd frontend-journey
mkdir part1-html part2-css part3-js part4-dom part5-async part6-tooling part7-react part8-integration

mkdir means “make directory” (create a folder) and cd means “change directory” (go into a folder). You just used your first real developer commands. Open the whole folder in VS Code: File → Open Folder → select frontend-journey.
