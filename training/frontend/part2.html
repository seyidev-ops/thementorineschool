2.1 What Is CSS, and Three Ways to Use It

CSS (Cascading Style Sheets) controls how HTML looks: colours, fonts, spacing, layout, animation. A CSS rule has a selector (what to style) and declarations (how to style it):

selector {
  property: value;
  another-property: another-value;
}

CSS can live in three places, but professionals use the third almost exclusively:

Inline — style="color:red" on a tag. Avoid: impossible to maintain.

Internal — a <style> block in the <head>. Fine for tiny experiments.

External — a separate .css file linked with <link rel="stylesheet" href="styles.css">. One stylesheet, many pages, one place to change everything. This is the professional default.

2.2 Selectors: Targeting What You Style

FILErt2-css/selectors.css
/* Element selector: every <p> on the page */
p { color: #333333; }
/* Class selector: anything with class="card" */
.card { background: white; }
/* ID selector: the ONE element with id="hero" */
#hero { text-align: center; }
/* Descendant: only <a> tags inside <nav> */
nav a { text-decoration: none; }
/* Pseudo-class: a link while the mouse hovers over it */
a:hover { color: #C8A04B; }
/* Multiple selectors share one rule */
h1, h2, h3 { font-family: Georgia, serif; }
THE CASCADE IN ONE SENTENCE When two rules conflict, the more specific selector wins (id beats class beats element), and if specificity ties, the rule written last wins. Ninety percent of “why isn't my CSS working?” moments are the cascade doing exactly what it was told.

2.3 The Box Model: The Most Important Idea in CSS

Every element on a page is a rectangular box made of four layers, from inside out: content, padding (space inside the border), border, and margin (space outside the border, pushing neighbours away).

FILErt2-css/box-model.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Box Model</title>
  <style>
    * { box-sizing: border-box; } /* professional default — width includes
                                     padding and border */
    .box {
      width: 300px;
      padding: 20px;
      border: 3px solid #0A0A0A;
      margin: 24px;
      background: #F3F0E9;
    }
  </style>
</head>
<body>
  <div class="box">Open DevTools (F12), click the inspector arrow, then
    click me. Look at the coloured box-model diagram in the Styles panel —
    that diagram is this lesson, live.</div>
  <div class="box">I am a second box. The gap between us is our margins.</div>
</body>
</html>
TRY THIS

Change padding to 0, then margin to 0, refreshing each time. Watching the boxes collapse teaches you more than any diagram.

2.4 Colour, Typography, and Units

ConceptWhat You Need to Know
ColoursUse hex codes (#C8A04B), rgb(200,160,75), or names (gold). Professionals define a small palette and reuse it.
font-familyAlways give fallbacks: font-family: Georgia, 'Times New Roman', serif;
pxFixed pixels. Fine for borders and small details.
remRelative to the root font size (usually 16px). 1.5rem = 24px. The professional unit for font sizes and spacing — it respects user accessibility settings.
%Relative to the parent element. width: 50% is half the parent's width.
vh / vw1% of the viewport height/width. height: 100vh = full screen height.

2.5 Flexbox: Arranging Things in a Row or Column

Flexbox solves the problem that tormented developers for fifteen years: placing boxes neatly beside each other and centring things. Turn any container into a flex container with display: flex, and its direct children line up automatically.

FILErt2-css/flexbox.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Flexbox</title>
  <style>
    .nav {
      display: flex;
      justify-content: space-between; /* horizontal distribution */
      align-items: center;            /* vertical alignment */
      background: #0A0A0A;
      padding: 16px 24px;
    }
    .nav a { color: white; text-decoration: none; margin-left: 16px; }
    .brand { color: #C8A04B; font-weight: bold; }
    .cards {
      display: flex;
      gap: 16px;          /* space BETWEEN children — no margin hacks */
      flex-wrap: wrap;     /* children wrap to the next line on small screens */
      padding: 24px;
    }
    .card {
      flex: 1;             /* each card grows to share space equally */
      min-width: 220px;
      border: 1px solid #DDD6C7;
      padding: 20px;
    }
  </style>
</head>
<body>
  <nav class="nav">
    <span class="brand">MY PORTFOLIO</span>
    <div>
      <a href="#">Home</a><a href="#">Projects</a><a href="#">Contact</a>
    </div>
  </nav>
  <div class="cards">
    <div class="card"><h3>Project One</h3><p>Description here.</p></div>
    <div class="card"><h3>Project Two</h3><p>Description here.</p></div>
    <div class="card"><h3>Project Three</h3><p>Description here.</p></div>
  </div>
</body>
</html>

Memorise the big four: display: flex, justify-content (main axis), align-items (cross axis), gap. They cover 90% of daily layout work. To centre anything perfectly: display: flex; justify-content: center; align-items: center; on the parent.

2.6 CSS Grid: Two-Dimensional Layouts

Flexbox arranges items along one axis; Grid arranges them in rows AND columns simultaneously — perfect for photo galleries, dashboards, and full page layouts.

FILErt2-css/grid.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Grid Gallery</title>
  <style>
    .gallery {
      display: grid;
      grid-template-columns: repeat(3, 1fr); /* three equal columns */
      gap: 12px;
      padding: 24px;
    }
    .gallery div {
      background: #F3F0E9;
      border: 1px solid #DDD6C7;
      padding: 40px 0;
      text-align: center;
    }
    .featured { grid-column: span 2; } /* this item is twice as wide */
  </style>
</head>
<body>
  <div class="gallery">
    <div class="featured">FEATURED</div>
    <div>2</div><div>3</div><div>4</div>
    <div>5</div><div>6</div><div>7</div>
  </div>
</body>
</html>

2.7 Responsive Design: One Site, Every Screen

Over 70% of web traffic in Africa is mobile. A site that breaks on a phone is a site that does not exist. Responsive design means your layout adapts to the screen, and the tool is the media query:

/* Mobile-first: write base styles for small screens, then
   add overrides as the screen grows. */
.cards { display: flex; flex-direction: column; gap: 16px; }
@media (min-width: 768px) {       /* tablets and up */
  .cards { flex-direction: row; }
}
@media (min-width: 1200px) {      /* large desktops */
  .cards { max-width: 1140px; margin: 0 auto; }
}
TEST LIKE A PROFESSIONAL In Chrome DevTools, click the phone/tablet icon (Ctrl+Shift+M) to toggle Device Mode. Drag the viewport, pick real devices from the dropdown, and watch your media queries fire. Do this on every page you ever build.

2.8 Transitions, Transforms, and CSS Variables

:root {                       /* define design tokens once */
  --ink: #0A0A0A;
  --gold: #C8A04B;
  --paper: #F3F0E9;
}
.btn {
  background: var(--ink);
  color: white;
  padding: 12px 28px;
  border: none;
  cursor: pointer;
  transition: all 0.25s ease;  /* animate every change smoothly */
}
.btn:hover {
  background: var(--gold);
  transform: translateY(-2px); /* lift the button slightly */
}

CSS variables (--name, read with var(--name)) let you change your entire site's colour scheme by editing three lines. Transitions make state changes feel premium instead of abrupt. Together they are the foundation of every polished interface.

2.9 Checkpoint Project: Style Your Profile Into a Portfolio

Take your Part 1 profile page and transform it into a portfolio you would not be embarrassed to send to an employer.

Requirements

Move all styling into an external styles.css. Define your palette with CSS variables in :root.

Sticky flexbox navigation bar with hover states on every link.

A full-height hero section (min-height: 100vh) with your name, title, and a call-to-action button.

A projects section using CSS Grid: three cards on desktop, one column on mobile (media query).

A styled contact form, with focus styles on the inputs (input:focus { border-color: var(--gold); }).

Smooth hover transitions on all buttons and cards.

Pass the phone test: nothing overflows, nothing is unreadable at 375px wide.
