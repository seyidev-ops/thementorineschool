8.1 How a Front End Talks to Everything Else

Here is the secret that demystifies all integration: the front end never cares what language the server is written in. Node, Python, PHP, Java, Go — they all speak to your JavaScript the same way: HTTP requests carrying JSON. If you can fetch(), you can integrate with anything. This part proves it three times over, then adds the tools modern teams use around that core.

8.2 JavaScript + Node/Express: Your Own API in 25 Lines

mkdir job-api && cd job-api
npm init -y
npm install express cors
FILEb-api/server.js
const express = require("express");
const cors = require("cors");
const app = express();
app.use(cors());          // allow your front end (other port) to call us
app.use(express.json());  // parse JSON request bodies
let jobs = [
  { id: 1, title: "React Developer", company: "TechCorp", remote: true },
  { id: 2, title: "UI Engineer", company: "FinPay", remote: false },
];
app.get("/api/jobs", (req, res) => res.json(jobs));
app.post("/api/jobs", (req, res) => {
  const job = { id: Date.now(), ...req.body };
  jobs.push(job);
  res.status(201).json(job);
});
app.delete("/api/jobs/:id", (req, res) => {
  jobs = jobs.filter(j => j.id !== Number(req.params.id));
  res.status(204).end();
});
app.listen(3000, () => console.log("API running on http://localhost:3000"));
node server.js   # then visit http://localhost:3000/api/jobs

Now point your React capstone's fetch at http://localhost:3000/api/jobs instead of the public API. You are officially a full-stack-capable front-end developer.

8.3 JavaScript + Python: The Power Couple

Python dominates data analysis and AI. Wrap any Python logic in a FastAPI server, and your front end consumes it like any other API.

Step 1 — A Python API With FastAPI

FILEalytics.py
# Install once:  pip install fastapi uvicorn
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
app = FastAPI()
# CORS: explicitly allow browser pages from other origins to call us
app.add_middleware(CORSMiddleware, allow_origins=["*"],
                   allow_methods=["*"], allow_headers=["*"])
@app.get("/api/salary-stats")
def salary_stats():
    salaries = [250000, 400000, 350000, 600000, 500000]
    return {
        "average": sum(salaries) / len(salaries),
        "highest": max(salaries),
        "lowest": min(salaries),
        "count": len(salaries),
    }
# Run with:  uvicorn analytics:app --port 8000

Step 2 — JavaScript Calls Python

// Anywhere in your front end — vanilla JS or a React useEffect:
const res = await fetch("http://localhost:8000/api/salary-stats");
const stats = await res.json();
console.log(`Average salary: ₦${stats.average.toLocaleString()}`);

Same fetch, same JSON, same pattern. The language behind the URL changed; your code did not. That is the lesson of this entire part.

8.4 JavaScript + PHP & WordPress: The Classic Web

Over 40% of all websites run WordPress (PHP). Modern front-end developers integrate with it through the WordPress REST API — every WordPress site exposes its posts as JSON at /wp-json/wp/v2/posts:

// Pull the latest posts from ANY public WordPress blog:
const res = await fetch(
  "https://techcrunch.com/wp-json/wp/v2/posts?per_page=3");
const posts = await res.json();
posts.forEach(p => console.log(p.title.rendered));

This pattern — a JavaScript front end consuming a CMS through its API — is called headless CMS architecture, and it is one of the most lucrative freelance niches: clients keep the WordPress admin they know, you deliver a fast modern React front end.

8.5 Integrating Payments: Paystack in the Front End

Real projects take money. Paystack's inline JavaScript shows the standard integration shape (test keys are free at paystack.com):

<script src="https://js.paystack.co/v1/inline.js"></script>
<script>
  function payNow() {
    const handler = PaystackPop.setup({
      key: "pk_test_xxxxxxxxxxxx",        // your PUBLIC test key
      email: "customer@email.com",
      amount: 500000,                       // amount in kobo = ₦5,000
      onClose: () => alert("Payment window closed."),
      callback: (response) =>
        alert("Payment complete! Ref: " + response.reference),
    });
    handler.openIframe();
  }
</script>
<button onclick="payNow()">Pay ₦5,000</button>
SECURITY RULE YOU MUST NEVER BREAK Only the PUBLIC key ever appears in front-end code. Secret keys live on a server, full stop. Payment verification also happens on the server — never trust the browser to confirm money moved.

8.6 WebAssembly: When JavaScript Needs a Turbocharger

WebAssembly (WASM) lets code written in Rust, C, or C++ run in the browser at near-native speed — powering Figma, AutoCAD Web, and in-browser video editing. You do not need to write WASM as a junior, but you should be able to say in an interview: “WASM modules are compiled from languages like Rust and called from JavaScript like any function, for compute-heavy work such as image processing — while JavaScript remains the orchestrator of the page.”
