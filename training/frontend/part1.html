1.1 What Is HTML?

HTML stands for HyperText Markup Language. It is not a programming language — it is a markup language, which means it describes structure, not behaviour. Think of building a house: HTML is the skeleton and walls, CSS (Part 2) is the paint and decoration, and JavaScript (Part 3 onwards) is the electricity and plumbing that makes things actually work.

Every web page you have ever visited — Google, Facebook, your bank — is built on HTML. When your browser loads a page, it reads an HTML file and draws what it describes.

1.2 Your First Web Page (Practical)

In VS Code, right-click the part1-html folder and choose New File. Name it index.html — exactly that, all lowercase.

Copy and paste the code below into the file, then press Ctrl+S (Cmd+S on Mac) to save.

Right-click anywhere inside the file and choose “Open with Live Server”. Your browser opens with your first web page.

FILErt1-html/index.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My First Web Page</title>
</head>
<body>
  <h1>Hello, World! I am becoming a front-end developer.</h1>
  <p>This is my very first web page, built with my own hands.</p>
</body>
</html>

Line-by-Line Breakdown

<!DOCTYPE html> — tells the browser “this is a modern HTML5 document”. Always the first line.

<html lang="en"> — the root container of everything. lang="en" tells screen readers and search engines the page is in English.

<head> — information ABOUT the page (title, character set). Nothing here is visible on the page itself.

<meta name="viewport" ...> — makes the page display properly on phones. Include it in every page you ever build.

<title> — the text shown in the browser tab.

<body> — everything visible lives here.

<h1> and <p> — a top-level heading and a paragraph.

THE ANATOMY OF A TAG Most HTML elements have an opening tag <p>, content, and a closing tag </p> with a forward slash. The browser reads <p>Hello</p> as “this is a paragraph containing the word Hello”. Forgetting a closing tag is the number one beginner bug.

1.4 Attributes: Giving Tags Extra Information

Attributes live inside the opening tag and are written as name="value". You have already used several: lang, href, src, and alt. Two more matter enormously from here on:

class — a reusable label you attach to any element so CSS and JavaScript can find it. Many elements can share one class.

id — a unique label for exactly one element on the page. No two elements may share an id.

<p class="warning">This paragraph is in the "warning" group.</p>
<p class="warning">So is this one.</p>
<p id="conclusion">There is only ONE conclusion on this page.</p>

1.5 Semantic HTML: Writing Pages That Mean Something

You could build an entire site with nothing but <div> tags, but professionals don't, because search engines, screen readers, and other developers cannot understand a wall of anonymous boxes. Semantic tags describe what each region of the page IS:

TagWhat It Marks
<header>Top region of the page or a section — usually logo and navigation
<nav>A group of navigation links
<main>The unique main content of the page (one per page)
<section>A themed grouping of content, usually with its own heading
<article>Self-contained content that makes sense alone (blog post, job listing, product card)
<aside>Side content: related links, adverts, callouts
<footer>Bottom region — copyright, contact, social links
WHY THIS MATTERS Semantic HTML is the difference between a page that ranks on Google and works for blind users, and a page that does neither. Interviewers check for it. It costs you nothing to do it right from day one.

1.6 Forms: Collecting Information From Users

Every login screen, search bar, and checkout page is a form. Forms are the front-end developer's bread and butter.

FILErt1-html/form-practice.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Registration Form</title>
</head>
<body>
  <h1>Course Registration</h1>
  <form>
    <label for="fullname">Full name</label><br>
    <input type="text" id="fullname" name="fullname" required><br><br>
    <label for="email">Email address</label><br>
    <input type="email" id="email" name="email" required><br><br>
    <label for="password">Password</label><br>
    <input type="password" id="password" name="password"
           minlength="8" required><br><br>
    <label for="track">Choose your track</label><br>
    <select id="track" name="track">
      <option value="frontend">Front-End Development</option>
      <option value="design">UI/UX Design</option>
      <option value="data">Data Analysis</option>
    </select><br><br>
    <label>
      <input type="checkbox" name="newsletter" checked>
      Send me weekly tips
    </label><br><br>
    <label for="bio">Tell us about yourself</label><br>
    <textarea id="bio" name="bio" rows="4" cols="40"></textarea><br><br>
    <button type="submit">Register</button>
  </form>
</body>
</html>

Key Points

Every <input> needs a <label>, connected by matching for and id values. Click the label — the input focuses. That is accessibility working.

type="email" and required give you free validation: the browser refuses to submit an empty or malformed email with zero JavaScript.

Other useful input types: number, date, tel, url, color, range, file. Change type="text" to each and watch the browser adapt.

1.7 Tables and Media

Tables are for tabular data (timetables, price lists, results) — never for page layout. Media tags embed images, audio, and video.

FILErt1-html/table-media.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Tables and Media</title>
</head>
<body>
  <h1>Weekly Study Timetable</h1>
  <table border="1">
    <thead>
      <tr><th>Day</th><th>Topic</th><th>Hours</th></tr>
    </thead>
    <tbody>
      <tr><td>Monday</td><td>HTML</td><td>2</td></tr>
      <tr><td>Tuesday</td><td>CSS</td><td>2</td></tr>
      <tr><td>Wednesday</td><td>Practice project</td><td>3</td></tr>
    </tbody>
  </table>
  <h2>Video</h2>
  <video src="demo.mp4" controls width="400">
    Your browser does not support video.
  </video>
  <h2>Audio</h2>
  <audio src="podcast.mp3" controls></audio>
</body>
</html>

1.8 Checkpoint Project: Your Personal Profile Page

Build a complete personal profile page from scratch — this becomes version one of your professional portfolio, and you will restyle and upgrade it throughout this guide.

Requirements

Use the full semantic skeleton: <header> with your name and a <nav> of links, <main> with at least three <section> blocks (About Me, Skills, Contact), and a <footer>.

Include a photo with a meaningful alt attribute.

Skills section: an ordered or unordered list of at least five skills you are building.

Contact section: a working form with name, email, message textarea, and a submit button.

Add a table showing your weekly study timetable.

Validate your page at https://validator.w3.org — fix every error it reports. Professionals ship valid HTML.
