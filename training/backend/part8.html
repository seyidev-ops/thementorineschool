8.1 HTTP Is the Lingua Franca

How does a Node API use a Python machine-learning model? How does Paystack tell your server a payment cleared? The same answer: services talk HTTP to each other, exactly the protocol you mastered in Part 1. A “microservice architecture” is simply several small servers being curl-style clients of one another. You already own every skill this part requires — now point them outward.

8.2 A Python Microservice, Consumed from Node (Practical)

Python dominates data work; Node dominates API work; professionals wire them together. Install Python from python.org if needed, then build a tiny analytics service with FastAPI:

cd backend-journey/part8-integration
pip install fastapi uvicorn
FILEpart8-integration/analytics.py
from fastapi import FastAPI

app = FastAPI()

@app.get("/stats")
def stats(amounts: str):
    nums = [float(x) for x in amounts.split(",") if x.strip()]
    if not nums:
        return {"error": "no numbers"}
    return {
        "count": len(nums),
        "total": sum(nums),
        "average": round(sum(nums) / len(nums), 2),
        "max": max(nums),
    }
uvicorn analytics:app --port 8000

Now a Node server that calls Python with the built-in fetch — your API composing another API:

FILEpart8-integration/node-client.js
import express from "express";
const app = express();

app.get("/report", async (req, res) => {
  const sales = [250000, 150000, 700000, 250000];
  const response = await fetch(`http://localhost:8000/stats?amounts=${sales.join(",")}`);
  const stats = await response.json();
  res.json({ source: "python-analytics", stats });
});

app.listen(4000, () => console.log("Node on :4000, consuming Python on :8000"));

Two terminals, two languages, one product: curl http://localhost:4000/report. (Alternative for scripts rather than services: Node's child_process.execFile("python3", ["script.py"]) runs Python directly and captures its output — handy for one-off jobs like PDF generation.)

8.3 Payments with Paystack (Practical)

Money is the most real-world integration there is. Paystack (the Nigerian standard, free test mode) follows the universal payment pattern: (1) initialize a transaction server-side with your secret key — never the browser — receiving a checkout URL; (2) the customer pays on Paystack's secure page; (3) verify server-side before delivering value. Create a free account at paystack.com, copy your test secret key into .env as PAYSTACK_SECRET, then:

FILEpart8-integration/payments.js
import "dotenv/config";
import express from "express";
const app = express();
app.use(express.json());

const PAYSTACK = "https://api.paystack.co";
const headers = {
  Authorization: `Bearer ${process.env.PAYSTACK_SECRET}`,
  "Content-Type": "application/json"
};

// Step 1: initialize
app.post("/pay", async (req, res) => {
  const { email, amountNaira } = req.body;
  const response = await fetch(`${PAYSTACK}/transaction/initialize`, {
    method: "POST",
    headers,
    body: JSON.stringify({ email, amount: amountNaira * 100 })   // kobo!
  });
  const data = await response.json();
  res.json({ checkout: data.data.authorization_url, reference: data.data.reference });
});

// Step 3: verify before delivering anything of value
app.get("/pay/verify/:reference", async (req, res) => {
  const response = await fetch(`${PAYSTACK}/transaction/verify/${req.params.reference}`, { headers });
  const data = await response.json();
  const paid = data.data && data.data.status === "success";
  res.json({ paid, amount: paid ? data.data.amount / 100 : 0 });
});

app.listen(4000, () => console.log("Payments demo on :4000"));

POST to /pay with your own email and amountNaira: 500, open the checkout URL, pay with Paystack's test card (4084 0840 8408 4081, any future date, CVV 408), then hit /pay/verify/... and watch paid: true come back. You have moved (test) money with your own code.

GOLDEN RULE

Trust only your server-side verify call. Never mark an order paid because the browser said so — browsers belong to customers, and some customers lie.

8.4 Webhooks: When the Internet Calls You Back

Verification by polling works, but production systems prefer webhooks: Paystack POSTs to your URL the instant a payment settles — the roles reverse and your API becomes the one receiving calls. A webhook endpoint is just a route: app.post("/webhooks/paystack", ...) that reads the event and updates your database. Two professional rules: respond 200 immediately (do slow work after responding), and verify the signature header (x-paystack-signature, an HMAC of the body with your secret) so pranksters cannot fake “payment received” calls. The same pattern powers GitHub pushes, WhatsApp messages and Stripe events — learn it once, integrate everything.

8.5 Email and File Uploads in Ten Minutes

Two integrations every product eventually needs. Email: npm install nodemailer, point it at any SMTP service (Gmail app-password for learning; Resend/Brevo in production) and transporter.sendMail({ to, subject, html }) — your Part 9 capstone sends order receipts this way. Uploads: npm install multer, add upload.single("photo") middleware to a route, and req.file appears with the stored file's details. Both are documented to death; the skill is recognising they are just middleware and async calls — shapes you already own.

8.6 Checkpoint Project: The Polyglot Pipeline

Wire one flow end-to-end: POST /orders (Node, JWT-protected) accepts an order → initializes a Paystack test payment → on verify, stores the order (SQLite) → calls the Python service to recompute sales stats → emails a receipt. Five technologies, one request chain, every link individually testable with curl. Draw the flow on paper first — architecture is a drawing skill before it is a typing skill. This pipeline is the spine of your capstone.
