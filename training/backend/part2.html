2.1 Why JavaScript on the Server?

JavaScript began in the browser, but Node.js (Part 3) lets it run anywhere — and it now powers the back ends of Netflix, PayPal, LinkedIn and Uber. For you, JavaScript means one language for your entire career surface: APIs, scripts, tooling, and (if you ever cross over) the front end too. Everything in this part runs in the terminal with node filename.js — no browser involved.

2.2 Variables: The Computer's Labelled Boxes

A variable is a labelled box in memory. const creates a box whose contents cannot be replaced (use it by default); let creates one that can. The data inside has a type: strings (text), numbers, booleans (true/false), or the special emptiness values null and undefined.

FILEpart2-js/01-variables.js
const schoolName = "The Mentorine School";   // string
const yearFounded = 2024;                     // number
let studentsEnrolled = 1280;                  // number that will change
const isAcceptingStudents = true;             // boolean

studentsEnrolled = studentsEnrolled + 1;      // allowed: let
// schoolName = "Other School";               // ERROR: const cannot be reassigned

console.log(`${schoolName} has ${studentsEnrolled} students.`);
console.log("Accepting new students:", isAcceptingStudents);

Run it with node 01-variables.js. The backtick string on the last lines is a template literal — ${...} inserts a value into text, and you will use it hundreds of times a week.

TRY THIS

Uncomment the schoolName reassignment line and run again. Read the error: “Assignment to constant variable.” You have just met your first runtime error — and survived.

2.3 Conditionals: Teaching the Computer to Decide

Back ends are decision factories: is this password correct? does this user have permission? is there stock left? Decisions are written with if, else if, and else, comparing values with === (equals), !== (not equals), > and <.

FILEpart2-js/02-atm.js
const balance = 45000;
const withdrawal = 50000;
const dailyLimit = 100000;

if (withdrawal > balance) {
  console.log("Insufficient funds. Your balance is " + balance);
} else if (withdrawal > dailyLimit) {
  console.log("Amount exceeds your daily limit of " + dailyLimit);
} else {
  const newBalance = balance - withdrawal;
  console.log(`Take your cash. New balance: ${newBalance}`);
}

Always use ===, never ==. The triple version refuses to compare apples with oranges; the double version silently converts types and causes bugs that take hours to find.

TRY THIS

Set withdrawal to 45000 exactly. Which branch runs? Now add a fourth rule: withdrawals must be multiples of 500 — the % (remainder) operator gives 0 when the division is clean: withdrawal % 500 === 0.

2.4 Loops: Doing Things Ten Thousand Times

A server that sends one email is a toy; a server that sends an email to every student is a product. Loops repeat work. The classic for loop counts; for...of walks through a collection.

FILEpart2-js/03-loops.js
// Classic counting loop
for (let i = 1; i <= 5; i++) {
  console.log(`Processing payment batch ${i} of 5`);
}

// Walking a collection
const students = ["Ada", "Femi", "Ngozi", "Tunde"];
for (const name of students) {
  console.log(`Sending welcome email to ${name}...`);
}
COMMON MISTAKE

Off-by-one errors: i <= 5 runs five times, i < 5 runs four. When a loop misbehaves, print i on every pass — console.log is the beginner's debugger, and honestly, the professional's too.

2.5 Functions: Reusable Machines

A function is a machine: ingredients in (parameters), result out (return value). Functions are how you stop repeating yourself and start building systems. Modern JavaScript writes them as arrow functions.

FILEpart2-js/04-functions.js
// A machine that calculates transaction fees: 1.5% capped at 2000
const calculateFee = (amount) => {
  const fee = amount * 0.015;
  return Math.min(fee, 2000);
};

// A machine that formats money
const naira = (amount) => `\u20a6${amount.toLocaleString()}`;

console.log(`Fee on ${naira(50000)}:  ${naira(calculateFee(50000))}`);
console.log(`Fee on ${naira(500000)}: ${naira(calculateFee(500000))}`);

Notice how naira and calculateFee compose — small machines plugging into each other. Professional back ends are thousands of small functions, each doing one thing well, not one giant scroll of instructions.

2.6 Arrays and Objects: The Data Shapes of Every API

Two shapes carry essentially all data on the internet. An array is an ordered list. An object is a labelled record of key: value pairs. Every JSON response you saw in Part 1 was just these two shapes nested inside each other — and every API you build will speak them.

FILEpart2-js/05-data-shapes.js
const course = {
  title: "Back-End Web Developer",
  durationWeeks: 16,
  priceNaira: 250000,
  topics: ["Node", "Express", "SQL", "MongoDB", "Security"]
};

console.log(course.title);          // dot access
console.log(course.topics[0]);      // first item of a nested array

// The three array superpowers of API work:
const prices = [250000, 500000, 700000];

const withVAT   = prices.map(p => p * 1.075);          // transform every item
const affordable = prices.filter(p => p <= 500000);     // keep some items
const revenue   = prices.reduce((sum, p) => sum + p, 0);// boil down to one value

console.log({ withVAT, affordable, revenue });
WHY THIS MATTERS

map, filter and reduce are 80% of real back-end data handling: “take these database rows, reshape them, drop the private fields, total the amounts.” Learn these three and API code starts reading like English.

2.7 Asynchronous JavaScript: The Restaurant Kitchen

A good kitchen never stands still waiting for the oven — the chef starts the soup while bread bakes. JavaScript works the same way: slow operations (database queries, network calls, file reads) are started, and the program continues; when the result arrives, your code resumes. This is asynchronous programming, and the modern syntax for it is async/await on top of promises.

FILEpart2-js/06-async.js
// A pretend slow operation, like a database query (1.5s)
const fetchBalance = (account) =>
  new Promise((resolve) => {
    setTimeout(() => resolve({ account, balance: 45000 }), 1500);
  });

const main = async () => {
  console.log("Requesting balance...");
  const result = await fetchBalance("0123456789");   // pause HERE only
  console.log(`Account ${result.account}: \u20a6${result.balance}`);
  console.log("Done.");
};

main();
console.log("(The kitchen keeps moving while we wait...)");

Run it and study the order of the output: the kitchen line prints before the balance arrives. await pauses only the function it lives in — the rest of your server keeps serving other customers. This single idea is why Node handles thousands of simultaneous users on one modest machine.

GOLDEN RULE

Anything touching a network, a disk, or a database is asynchronous. If a value seems mysteriously “undefined”, you almost certainly forgot an await.

2.8 Checkpoint Project: The Bank Account Simulator

Build a terminal program that simulates a bank account using everything from this part: an object for the account, functions for deposit/withdraw/statement, validation with conditionals, a loop to process a batch of transactions, and an async “processing” delay for realism.

FILEpart2-js/checkpoint-bank.js
const account = { owner: "Ada Obi", balance: 100000, history: [] };

const delay = (ms) => new Promise(r => setTimeout(r, ms));

const deposit = (amount) => {
  if (amount <= 0) return "Deposit must be positive.";
  account.balance += amount;
  account.history.push({ type: "deposit", amount });
  return `Deposited \u20a6${amount}. New balance: \u20a6${account.balance}`;
};

const withdraw = (amount) => {
  if (amount > account.balance) return "Insufficient funds.";
  account.balance -= amount;
  account.history.push({ type: "withdraw", amount });
  return `Withdrew \u20a6${amount}. New balance: \u20a6${account.balance}`;
};

const statement = () =>
  account.history.map((t, i) => `${i + 1}. ${t.type} \u20a6${t.amount}`).join("\n");

const main = async () => {
  const batch = [
    () => deposit(50000), () => withdraw(20000),
    () => withdraw(999999), () => deposit(-5)
  ];
  for (const transaction of batch) {
    await delay(700);                 // pretend processing time
    console.log(transaction());
  }
  console.log("\n--- STATEMENT for " + account.owner + " ---");
  console.log(statement());
};

main();

Upgrade challenges (do at least two): add a transfer function that moves money between two account objects; reject withdrawals over a daily limit by summing today's history with reduce; add a timestamp to every history entry with new Date().toISOString().
