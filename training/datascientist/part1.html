1.1 Variables and Types

Python is the language of data science because it reads like pseudocode and carries the world's best scientific libraries. Run files with python filename.py inside part1-python:

FILEpart1-python/01_basics.py
bank = "NaijaTrust"          # string
customers = 12500             # int
default_rate = 0.043          # float
is_regulated = True           # bool

print(f"{bank}: {customers:,} customers, default rate {default_rate:.1%}")

The f-string with {value:format} is your reporting voice: :, adds thousand separators, :.1% renders a clean percentage. You will use both daily.

1.2 Decisions: if / elif / else

FILEpart1-python/02_decisions.py
income = 180000
existing_loans = 1
credit_score = 645

if credit_score < 500:
    decision = "Declined: score too low"
elif income < 100000:
    decision = "Declined: income below threshold"
elif existing_loans >= 3:
    decision = "Manual review: heavy existing exposure"
else:
    decision = "Approved"

print(decision)

Read top to bottom: the first true branch wins. This little ladder is, structurally, a hand-written model — Part 6 will teach machines to learn such ladders from ten thousand examples instead.

1.3 Loops and Functions

FILEpart1-python/03_loops_functions.py
def monthly_payment(principal, annual_rate, months):
    """Flat-rate loan: principal plus simple interest, split evenly."""
    total = principal * (1 + annual_rate * months / 12)
    return total / months

loans = [200000, 500000, 1500000]
for amount in loans:
    pay = monthly_payment(amount, 0.24, 12)
    print(f"\u20a6{amount:,} loan -> \u20a6{pay:,.0f}/month")

A function is a named machine: inputs in, result out, documented by its docstring. Data science is thousands of small honest machines composed into pipelines.

1.4 Lists and Dictionaries: The Shapes of Records

FILEpart1-python/04_shapes.py
customer = {"name": "Ada Obi", "income": 180000, "score": 645, "loans": 1}
print(customer["name"], "earns", customer["income"])

portfolio = [
    {"name": "Ada",   "score": 645, "defaulted": False},
    {"name": "Femi",  "score": 480, "defaulted": True},
    {"name": "Ngozi", "score": 720, "defaulted": False},
    {"name": "Tunde", "score": 505, "defaulted": True},
]

# list comprehensions: filter and transform in one readable line
risky = [c["name"] for c in portfolio if c["score"] < 550]
avg_score = sum(c["score"] for c in portfolio) / len(portfolio)
default_rate = sum(c["defaulted"] for c in portfolio) / len(portfolio)

print("risky:", risky)
print(f"avg score {avg_score:.0f}, default rate {default_rate:.0%}")
WHY THIS MATTERS

A list of dictionaries IS a dataset — rows and columns in their rawest form. Notice defaulted (True/False) summed as numbers: True counts as 1. That tiny trick computes every rate you will ever report.
